This page explains the basics of the GitHub workflow.

What is a commit?

When changes are made to a file and added to a branch, this is a commit. Using commits is useful as it helps to keep track of changes made and the progress of a branch. Using commits means it is easy to look back and see what has been done, making it easier for others to understand the process - this is particularly useful for team working! Commits also make it easier to see where errors or bugs may have occured.

Note - the commit message makes a note of what changes have been made and why they were made.

What is a branch?

Branches are a way of seperating ideas from the main branch of a project. New branches are places where new ideas can be tested without affecting the main branch until you (and your team) are happy with it.

Note - branch names should be clear and descriptive!

What is a pull request?

A pull request allows other members of the team to see your code before it is added to the main branch, they can also provide advice or offer suggestions for changes.

Note - pull requests can be used at any time whilst working on an aspect of your project, not just when it's finished.
